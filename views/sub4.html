Preference of Website Technologies

Justification of Website Creation Technologies

Every developer has their strengths when it comes to programming and web development, that is why I have curated a list of my preferences for tools and techniques that are commonly utilized within web development and programming alike. Along with each prefered tool and/or technique I will provide a jusification for this preference.

Tools and Software

Below is a list of programming languages (or markdown languages) that I prefer to utilize. Note: React.js is a JavaScript framework that allows for a dynamic development environment.
  • HTML, CSS
  • JavaScript
  • React.js
  • Scss/Sass
Justification

HTML,CSS and JavaScript all go hand-in-hand and is the simplest form of web development tool (for myself), consists of all required functionalities for adequate user experiences, whilst not overcomplicating concepts. Finding a suitable platform build your website on is key and this is why I prefer to use HTML,CSS and JavaScript for web development.

React.js and Scss/Sass take what is already good about CSS and JavaScript and add even more functionality to it. This extra functionality may be key for the web application to achieve a certain degree of user experience etc. React.js allows JavaScript to interface with more points of entry within the browser and appends more functionality onto the codebase, which means React.js may be able to execute instructions that would otherwise seem "complex" or "very inefficient" inside of JavaScript.

Scss/Sass had already been explained in the previous section. Below is a list of software(s) that I commonly utilize.
  • IDE; Visual Studio Code
  • Chrome & Chrome DevTools
  • XAMPP
  • GitHub
Justification

Programming languages have many working parts, which are hard to distinguish without the help of an IDE. In laymans terms; an IDE provides a platform that can be utilized to view files, which it will then apply stlyes to the content of that file. These styles express what the content, that is being styled, represents. Please look below for an example of code inside of Visual Studio Code, notice how certain words are highlighted with color and text styles such as indentations are applied. They are applied for a purpose, for user readability.

Chrome and Chrome DevTools are the only form of output that you have when debugging on the client, Fortunatley the DevTools environment is very well developed and has many features that serve to be extremely useful and effective. DevTools allows you to see console output, network communications from the client, website sources, debugging functionalities, locally stored browser data, peformance of specified code blocks, and much more.

I use XAMPP to simulate a server environment where web files would be hosted, which then anyone can view from anywehere in the world. This is useful because it simulate the "real thing", the actual environment that your website files will reside in once you actually push your website to a web server. Additionaly, you are able to use XAMPP to host web files so that other users can develop on the same files simultaneously. XAMPP is an excellent tool to use for web development.

GitHub is an embodiment of Push and Pull architectures, where you (the developer) will push your code changes to a remote server which then saves that response for the next developer that comes along. This next developer can then request the latest version of the file and then push changes when finished. Utlizing this architecture for web development enables multiple developers to develop the same website simultaneously.
